What Are AI Agents?

An AI Agent is a system that uses an LLM as its core reasoning engine to autonomously decide what actions to take and in what order, in order to accomplish a user-defined goal.

Chatbot vs. Agent

Chatbot

  • Reactive — responds to prompts
  • Single turn or simple multi-turn
  • No tool use
  • No planning / reasoning loop

Agent

  • Proactive — plans & acts
  • Multi-step reasoning
  • Uses tools & APIs
  • Observe → Think → Act loop

The Agent Loop

🧑 User Goal
→
🧠 LLM Reasoning
→
🔧 Tool / Action
→
👀 Observation
→
🔁 Repeat or ✅ Done

Also known as the ReAct (Reasoning + Acting) loop

Core Components of an Agent

🧠 LLM (Brain)

The foundation model (GPT-4o, Claude, Gemini, Llama, etc.) that powers reasoning, planning, and language understanding.

🔧 Tools

External capabilities the agent can invoke: web search, code execution, database queries, API calls, file I/O, etc.

🗂️ Memory

Short-term: conversation context.
Long-term: vector stores, knowledge bases, or persistent state across sessions.

📋 Planning / Orchestration

Strategy for decomposing goals into sub-tasks, choosing tools, handling errors, and deciding when to stop.

Memory feeds context back into the LLM for subsequent reasoning steps

Agentic Design Patterns

Single Agent

One LLM handles all reasoning, tool calls, and decisions in a single loop. Simplest to build but limited in complexity. 
User → [Agent + Tools] → Response

Multi-Agent

Multiple specialized agents collaborate. Each has its own system prompt, tools, and responsibilities. A supervisor or router delegates tasks. 
User → Supervisor Agent
            ├── Research Agent (web search)
            ├── Coder Agent (code gen)
            └── Reviewer Agent (validation)

Other Patterns

  • Reflection: Agent critiques its own output and iterates
  • Plan-and-Execute: Separate planning step, then sequential execution
  • Tool-use / Function Calling: LLM selects from structured tool definitions
  • RAG + Agent: Retrieval-augmented generation as a tool inside the agent loop
  • Human-in-the-loop: Agent pauses for user approval at critical steps

Popular Agent Frameworks

Framework By Language Highlights
LangGraph LangChain Python / JS Graph-based orchestration, cycles, persistence, human-in-the-loop
CrewAI CrewAI Inc. Python Role-based multi-agent teams, built-in task delegation
AutoGen Microsoft Python / .NET Multi-agent conversations, code execution, extensible
OpenAI Agents SDK OpenAI Python Lightweight, built-in tracing, handoffs, guardrails
Semantic Kernel Microsoft C# / Python / Java Enterprise-ready, plugins, planners, Azure integration
Haystack deepset Python Pipeline-based, strong RAG support, modular

Choosing a Framework

  • Simple single agent? → OpenAI Agents SDK or LangGraph
  • Multi-agent teams? → CrewAI or AutoGen
  • Enterprise / .NET? → Semantic Kernel or AutoGen
  • Graph-based workflows? → LangGraph
  • RAG-heavy pipelines? → Haystack

Building an Agent

Code Walkthrough with OpenAI Agents SDK

1 — Install

pip install openai-agents

2 — Define Tools

from agents import function_tool

@function_tool
def get_weather(city: str) -> str:
    """Get current weather for a city."""
    # In production, call a real weather API
    return f"The weather in {city} is 72°F and sunny."

@function_tool
def search_web(query: str) -> str:
    """Search the web for information."""
    # In production, call a search API
    return f"Top result for '{query}': ..."

3 — Create the Agent

from agents import Agent

agent = Agent(
    name="Assistant",
    instructions="""You are a helpful assistant.
    Use available tools to answer questions accurately.
    Always cite your sources when using search.""",
    tools=[get_weather, search_web],
    model="gpt-4o",
)

4 — Run the Agent

from agents import Runner

result = Runner.run_sync(
    agent,
    "What's the weather in San Francisco?"
)

print(result.final_output)
# → "The weather in San Francisco is 72°F and sunny."

5 — Multi-Agent Handoff

from agents import Agent

# Specialist agents
weather_agent = Agent(
    name="Weather Agent",
    instructions="You handle weather queries only.",
    tools=[get_weather],
)

research_agent = Agent(
    name="Research Agent",
    instructions="You handle research queries only.",
    tools=[search_web],
)

# Triage agent delegates to specialists
triage_agent = Agent(
    name="Triage Agent",
    instructions="""Determine the user's intent and 
    hand off to the appropriate specialist.""",
    handoffs=[weather_agent, research_agent],
)

result = Runner.run_sync(
    triage_agent,
    "What's the weather like in Tokyo?"
)
print(result.final_output)

Deploying Agents

Deployment Options

🐳 Containers

Package agent as a Docker container. Deploy on Kubernetes, ECS, or Azure Container Apps for scalability.

⚡ Serverless

AWS Lambda, Azure Functions, or GCP Cloud Functions. Great for event-driven agents with sporadic usage.

☁️ Managed Platforms

LangSmith, Azure AI Agent Service, AWS Bedrock Agents. Hosting, tracing, and monitoring included.

🖥️ API Server

Wrap agent in FastAPI / Flask. Expose REST or WebSocket endpoints. Deploy on any VM or PaaS.

Example: FastAPI + Docker

# app.py
from fastapi import FastAPI
from agents import Agent, Runner

app = FastAPI()

agent = Agent(
    name="Assistant",
    instructions="You are a helpful assistant.",
    tools=[get_weather, search_web],
)

@app.post("/chat")
async def chat(user_message: str):
    result = await Runner.run(agent, user_message)
    return {"response": result.final_output}

Dockerfile

FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]

# Build & run
docker build -t ai-agent .
docker run -p 8000:8000 -e OPENAI_API_KEY=$KEY ai-agent

Production Checklist

  • ✅ Observability — tracing, logging, metrics (LangSmith, OpenTelemetry)
  • ✅ Guardrails — input/output validation, content filters
  • ✅ Rate limiting — protect LLM API costs
  • ✅ Error handling — retries, fallbacks, circuit breakers
  • ✅ Security — secrets management, least-privilege tool access
  • ✅ Evaluation — automated evals, A/B testing

Key Takeaways

  1. Agents = LLM + Tools + Memory + Planning
  2. The ReAct loop is the fundamental pattern
  3. Choose a framework that fits your complexity & ecosystem
  4. Start simple (single agent) → evolve to multi-agent when needed
  5. Treat deployment like any production service — observability, security, evals
